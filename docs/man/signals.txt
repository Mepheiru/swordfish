Signal Control -- How Swordfish Sends Signals -- Compiled for v1.2.5

Swordfish gives you full control over which signal gets sent to matching processes.

  -k                     Send SIGTERM (graceful shutdown — process can clean up)
  -K                     Send SIGKILL (forceful shutdown — process is terminated immediately)

Think of "-k" as asking nicely and "-K" as not asking at all.

You can also send any arbitrary signal using "-<sig>". Both numeric and named forms work:

  -9                     Send SIGKILL (same as -K)
  -15                    Send SIGTERM (same as -k)
  -10                    Send SIGUSR1
  -TERM                  Send SIGTERM by name
  -KILL                  Send SIGKILL by name
  -HUP                   Send SIGHUP (commonly used to reload configs)

Supported named signals: HUP, INT, QUIT, KILL, TERM, USR1, USR2, STOP, CONT

If you group "-k" and "-K" together (e.g. "-kK"), the last one wins, so "-kK" sends SIGKILL.
