Signal Control -- How Swordfish Sends Signals -- Compiled for v1.3.0

Swordfish gives you full control over which signal gets sent to matching processes.
The signal embeds directly after -k as a number or named signal.

  -k                     Send SIGTERM (default — graceful shutdown)
  -k9                    Send SIGKILL (forceful — process is terminated immediately)
  -k15                   Send SIGTERM explicitly by number
  -kHUP                  Send SIGHUP (commonly used to reload configs)
  -kUSR1                 Send SIGUSR1
  -kUSR2                 Send SIGUSR2

Think of "-k" as asking nicely and "-k9" as not asking at all.

Named signals can be combined with modifiers. The signal always comes immediately
after -k, modifiers follow after:

  -k9y firefox           SIGKILL + auto-confirm
  -kHUPy nginx           SIGHUP + auto-confirm
  -k9xy firefox          SIGKILL + exact match + auto-confirm

Supported named signals: HUP, INT, QUIT, KILL, TERM, USR1, USR2, STOP, CONT

If no signal is specified, SIGTERM is used. To escalate to SIGKILL automatically
if the process doesn't die, use --timeout:

  -k --timeout 5 firefox   Send SIGTERM, escalate to SIGKILL after 5 seconds
