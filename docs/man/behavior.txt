Behavior -- Confirmation and Execution -- Compiled for v1.3.0

When Swordfish matches processes, it shows them and asks for confirmation before
sending any signals. The flags below change how that works.

  -y                     Auto-confirm; skip the confirmation prompt
  -S                     Interactive selection — choose which matched processes to act on
  -t                     Always act on only the top (first) matched process
  -n                     Dry run — show what would happen without doing it
  -w                     Wait for the process to die after sending a signal
  --retry <seconds>      Retry every <seconds> if no processes are matched
  --timeout <seconds>    Escalate to SIGKILL after <seconds> if process does not die

Static Mode:
Running Swordfish with just a pattern and no operation flag enters Static Mode.
No signals are sent and it just lists matching processes. Good for checking what's
running before committing to a kill.

Verbosity:
The "-v" flag increases how much info is shown per process. Stack up to three:

  0 (default)
    PID "<process>" owned by <user>

  1 (-v)
    PID "<process>" cmdl: "<args>" threads: "<n>" owned by <user>

  2 (-vv)
    [<state>] PID "<process>" cmdl: "<args>" threads: "<n>" owned by <user>

  3 (-vvv)
    [<state>] PID "<process>" cmdl: "<args>" threads: "<n>" owned by <user>
        RAM: "<n> MB" AGE: "<n> min"

Hooks:
Script hooks let you run a file before or after signals are sent. Useful for
notifications, logging, or anything else you want to automate around a kill.

  --pre-hook <script>    Run <script> before sending signals
  --post-hook <script>   Run <script> after sending signals

The script receives the PID and process name as arguments.
